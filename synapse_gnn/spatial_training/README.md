Spatial Training Pipeline for Connectomics Edge Classification

This module provides a robust, spatially-aware training and evaluation pipeline for classifying synapses in large-scale connectomics volumes. It uses a GraphSAGE-based architecture to differentiate true biological synapses from structural false positives (candidates).

Overview

Unlike standard random node/edge splits, this pipeline utilizes spatial masking to ensure that training and testing sets represent distinct, biologically dense physical regions of the brain volume. It is designed to evaluate both Euclidean-based proximity graphs (baseline) and Axon-Dendrite Proximity (ADP) graphs containing tens of millions of hard negative candidates.

1. Prerequisites & Data Requirements

Ensure your target data directory (e.g., the shared project folder) contains the following generated artifacts before starting:

x_features.pt: The morphological feature tensor for all neurons. (Generated via preprocessing.py)

node_mapping.json: The mapping of biological 18-digit IDs to tensor indices. (Generated via preprocessing.py)

synapses.json: Ground truth synapse annotations. (Generated via teammate's script)

adp_base_edges.pt (or euc_base_edges.pt): The raw candidate edges generated by the graph adapter. (Generated via convert_adp_dict.py or build_euc_graph.py)


A Note on Storage: Ensure your shared filesystem has adequate quota (at least 20-50GB of free space) before running the split and stitch operations, as PyTorch requires significant overhead to save large tensors.

2. Configuration (config.json)

The entire pipeline is config-driven. Before running any scripts, verify your config.json.

Sample config.json Template

{
  "paths": {
    "data_dir": "/p/mlatuva/jhu-graph/cache_spatial",
    "model_save_dir": "./saved_models_spatial"
  },
  "spatial_split": {
    "train_cluster_size": 35000,
    "test_cluster_size": 20000,
    "spatial_threshold_nm": 50000
  },
  "training": {
    "learning_rate": 0.001,
    "epochs": 50,
    "steps_per_epoch": 500,
    "train_node_sample_size": 6000,
    "test_node_sample_size": 6000
  }
}


Key parameters to adjust based on the graph type:

ADP Graph Run (High Density): Set spatial_threshold_nm to 50000. Increase train_cluster_size (e.g., 35000) and test_cluster_size (e.g., 20000) to capture enough internal ground-truth synapses.

Euclidean Baseline Run: Set spatial_threshold_nm to 20000. Smaller cluster sizes (e.g., 10000) are usually sufficient due to the dense, localized nature of purely distance-based edges.

3. Pipeline Execution Steps

Step 1: Generate the Spatial Split

python spatial_training/create_spatial_split.py --config config.json


What this does:
Calculates the physical "Center of Mass" of all ground truth synapses by mapping their connected neurons to the feature space. It then anchors the Train and Test seeds slightly offset from this density center and uses k-Nearest Neighbors (k-NN) to grow masks. This guarantees that both sets contain high concentrations of true positives.

Output: spatial_split_masks.pt

Step 2: Split and Stitch Edges

python spatial_training/split_and_stitch_edges.py --config config.json


What this does:
Applies the spatial masks to both the ground truth synapses and the structural candidates. It requires both the source and destination neurons of an edge to fall within the same spatial mask to be included. This creates the final, cleanly separated positive and negative sets for the model.

Outputs: graph_train_edges.pt, graph_test_edges.pt, graph_train_spatial_candidates.pt, graph_test_spatial_candidates.pt

Step 3: Model Training and Evaluation

python spatial_training/train_and_eval.py --config config.json 2>&1 | tee training_run.log


What this does:
Initializes the GraphSAGE model and dynamically samples subgraphs from the stitched files. It utilizes hard negative mining, forcing the model to distinguish between true synapses and structurally similar candidates (especially critical in the ADP runs).

Outputs: best_model_[type].pth, metrics_[type].json

4. Troubleshooting

Empty Tensor / ValueError during Training: This indicates your spatial split failed to capture enough ground truth synapses. Re-run create_spatial_split.py with larger cluster_size values in your config.

Out of Memory (OOM) Errors: If portal node VRAM is exhausted, lower the train_node_sample_size in your config.json.

File Save Failures / Corrupted .pt files: Check your filesystem quota (df -h). PyTorch will silently fail or write truncated files if the disk is at 100% capacity.
